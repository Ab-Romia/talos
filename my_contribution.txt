File Upload & Processing System

Built the backend system that lets users upload, download, and manage files within their workspaces.

1. Set up MinIO as the file storage server, integrated it with the existing database to track file metadata (name, size, type, status, who uploaded it, etc.)

2. Created the API endpoints for uploading, downloading, listing, and deleting files, as well as attaching files to chat messages.

3. Added access control so only workspace members can interact with files in their workspace.

4. Built a background worker that automatically processes uploaded files:
   a. Documents (PDFs, Word files, etc.) get parsed and split into text chunks, ready for the AI search/retrieval feature.
   b. Images get thumbnails generated for previews.

5. The background processing runs on a separate queue (Redis + ARQ) so it doesn't slow down the main app.

6. Everything ties into the existing user, workspace, and chatroom structure.
