## ARCHITECTURE AND TECHNICAL DECISIONS
The system implements a modular RAG architecture with six main components: indexing, pre-retrieval, retrieval, post-retrieval, generation, and orchestration. We selected OpenAI embeddings (text-embedding-3-small) for semantic search due to their superior quality and cost-effectiveness. The reranking module uses cross-encoder models (ms-marco-MiniLM-L-6-v2) which improved relevance scores by 23% in our benchmarks. Query routing classifies questions into four types (factual, complex, conversational, unclear) and adapts the pipeline accordingly. Conversation memory tracks up to 10 turns by default, enabling context-aware multi-turn dialogues. The system supports YAML-based configuration for easy customization without code changes.

## TECHNICAL WALKTHROUGHS
How to add a new retrieval module: Create a class in modules/retriever/ implementing the retrieve() method with query and top_k parameters. Update __init__.py to export the new class. Add configuration options in config/rag_config.yaml under the retriever section. Test with the existing pipeline.
Setting up the development environment: Clone repository, create Python 3.9+ virtual environment, install dependencies with pip install -r requirements.txt, copy .env.example to .env and add your OPENAI_API_KEY. Run python main.py to start in modular mode.
Running the web interface: Activate virtual environment, ensure .env has API key configured, run python app.py to start FastAPI server on port 8000. Access the interface at http://localhost:8000 in your browser.
Adding new knowledge to the workspace: Click the edit button (pencil icon) in the web interface, modify the content in the text editor, save changes, then re-index the knowledge base to apply updates.

## EVALUATION METRICS AND BENCHMARKS
We maintain a test dataset of 100 question-answer pairs covering factual queries, complex questions, conversational follow-ups, and edge cases.
Retrieval metrics: Mean Reciprocal Rank (MRR) measures how quickly correct documents appear, Normalized Discounted Cumulative Gain (NDCG) evaluates ranking quality, Recall@K for K=3,5,10 checks coverage.
Generation metrics: BLEU score for n-gram overlap, ROUGE-L for longest common subsequence, human evaluation for faithfulness and relevance.
Reranker performance: Cross-encoder reranking improved MRR from 0.67 to 0.82 (23% improvement). Processing time increased by 150ms but accuracy gains justify the latency.
Current baseline: Simple RAG achieves 0.65 MRR, Modular RAG with reranking achieves 0.82 MRR. Conversation memory adds 15% improvement on follow-up questions.

## ISSUES AND BLOCKERS - RESOLVED
Issue: Line-by-line chunking caused poor context retrieval - RESOLVED by implementing section-based chunking using headers.
Issue: High temperature (0.7) caused hallucinations in responses - RESOLVED by reducing to 0.1 and strengthening system prompts.
Issue: Conversation history not being utilized effectively - RESOLVED by passing last 3 turns to generator and implementing context resolution.
Issue: Reranking adding too much latency - OPTIMIZED by reducing candidate set and using faster cross-encoder model.

## WORKSPACE FEATURES
Multi-turn conversations: System maintains conversation history for context-aware responses. Users can reference previous questions with pronouns and context.
Knowledge base editing: Web interface allows direct editing of workspace knowledge through modal editor. Changes take effect after re-indexing.
Progress tracking: Real-time progress indicators show indexing steps with detailed status messages. Users see exactly where processing is happening.
Pipeline visualization: Web interface displays active pipeline components and their configuration status.
Query routing: System automatically detects query type and adapts retrieval strategy for optimal results.
Conversation management: Special commands (/clear, /stats, /history) provide control over conversation state.
